# ST_FOC 修改記錄準則

每次對專案任何檔案（.c / .h / .md 等）做修改後，**必須**同步更新 `d:\Work_202506\FOC\ST_FOC\ST_FOC\devlog.md`。

## 記錄格式

在 devlog.md 頂部（第一條 `---` 之後）插入新條目，格式如下：

```markdown
## YYYY-MM-DD HH:MM

### [修改標題]

**原因：** 說明為什麼要做這個修改。

**目的：** 說明這個修改要達成什麼效果。

**修改方式：** 列出修改的檔案與具體變更：
- `檔案路徑` — 修改內容摘要

**日期：** YYYY-MM-DD HH:MM:SS

---
```

## 注意事項

1. 每次修改**至少一條**記錄，不可省略
2. 若同一時間有多個相關修改，可合併成一條記錄
3. 使用繁體中文撰寫
4. 「原因」說明觸發原因，「目的」說明預期效果，「修改方式」列具體檔案與變動
